# ezmirror — Agent Guide

## Commands

```bash
cargo build --release           # daemon binary → target/release/ezmirord
make build                       # build + copy to repo root
cargo clippy                     # lint Rust
python3 -m py_compile web/panel.py python/setup.py  # Python syntax check
sudo bash setup.sh               # full install (interactive)
sudo bash setup.sh --unattended  # unattended install
python3 web/panel.py             # start admin panel in dev (port 8080)
```

## Architecture

**Three layers, all behind a single nginx:**

| Layer | Entrypoint | Function |
|---|---|---|
| nginx | `/etc/nginx/sites-available/default` | Serves `/{slug}/` (fancyindex), proxies `/admin/` → `:8080`, `/healthz` → `:9633` |
| Rust daemon | `src/main.rs` | Sync loop, Prometheus metrics (`:9633/metrics`), health check, `--sync`/`--daemon`/`--status` flags |
| Admin panel | `web/panel.py` | FastAPI at `:8080`, proxied at `/admin/`, CRUD mirrors, view logs/metrics |

CLI flags: `--daemon`, `--sync`, `--sync-slug <slug>`, `--dry-run`, `--status`, `--port <n>`.

**No subcommands** — flat flags only.

## Config — Two Sources of Truth

| File | Format | Read by | Purpose |
|---|---|---|---|
| `mirrors.json` | JSON array | setup.py, panel.py | Catalog — single source of truth |
| `/etc/ezmirror/mirrors.conf` | pipe-delimited | ezmirord (daemon) | Active mirrors — generated from mirrors.json |

Daemon only reads `mirrors.conf`. Panel writes both when creating/deleting mirrors via API.

nginx config is generated by `setup.py` with per-mirror `location /{slug}/` blocks. Panel API inserts/removes location blocks by string manipulation into the existing config, then reloads nginx.

## Key Non-Obvious Facts

- **Admin HTTP auth is at nginx level**, not in the app. Credentials in `/etc/ezmirror/.htpasswd` (managed by setup.py, env vars `EZMIRROR_ADMIN_USER`/`EZMIRROR_ADMIN_PASS`).
- **Docker**: entrypoint runs `/opt/ezmirror/web/panel.py` directly (no systemd). Sets `EZMIRROR_SKIP_DEPS=1` — skips apt/pip installs. The daemon is pre-built in the image (multi-stage).
- **Release**: tagging `v*` triggers CI. Tag version must match `VERSION` file or the workflow fails.
- **Mirror paths**: flat at `/{slug}/` — no `/pub/` prefix. This is enforced everywhere (nginx, templates, config parser).
- **`mirrors.conf` pipe format**: `slug|Name|Description|upstream|method|size|warn|interval|bandwidth|retention_days|retention_max_gib`
- **Admin panel API**: `POST/DELETE /admin/api/mirrors` creates/removes mirrors (writes all configs, inserts nginx location, reloads nginx). The create endpoint accepts a `MirrorCreate` pydantic model.
- **`setup.sh` is the entry point**, not `setup.py` directly. It pre-flights (curl, git, jq, rsync), forwards env vars, runs setup.py, then builds the daemon with `make build`.
- **Python deps**: fastapi + uvicorn (for panel), installed by setup.py via pip.
- **Rust deps**: clap (derive), serde (derive), serde_json, libc. Release profile has LTO + strip.

## Conventions

- Git: linear history on main, squash feature branches
- Config backward compat: any new field must not break existing `mirrors.conf` parser (the daemon reads by positional index, not header)
- JSON API responses use camelCase
