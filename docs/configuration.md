# Configuration

## Mirror Catalog (`mirrors.json`)

Single source of truth for all mirror definitions. Located at the project root.

```json
{
  "slug": "debian",
  "name": "Debian GNU/Linux",
  "description": "Stable, testing, and unstable package archive",
  "upstream": "rsync://rsync.debian.org/debian/",
  "method": "rsync",
  "size": "~2.0 TiB",
  "warn": "large",
  "interval": "12h",
  "bandwidth": 0,
  "retention_days": 0,
  "retention_max_gib": 0
}
```

| Field | Required | Description |
|-------|----------|-------------|
| `slug` | yes | URL path segment (e.g., `debian` → `/{slug}/`) |
| `name` | yes | Human-readable distribution name |
| `description` | yes | Short description |
| `upstream` | yes | rsync/rclone URL |
| `method` | no | `rsync` or `rclone` (default: `rsync`) |
| `size` | no | Approximate size for display |
| `warn` | no | Warning label (`large`) |
| `interval` | no | Sync interval (`6h`, `12h`, `1d`) |
| `bandwidth` | no | Bandwidth limit in KB/s (0 = unlimited) |
| `retention_days` | no | Days of version retention (0 = unlimited) |
| `retention_max_gib` | no | Max GiB of version retention (0 = unlimited) |

## Active Mirrors (`/etc/ezmirror/mirrors.conf`)

Generated from `mirrors.json` by setup.py. Pipe-delimited format for the daemon:

```
slug|Name|Description|upstream|method|size|warn|interval|bandwidth|retention_days|retention_max_gib
debian|Debian GNU/Linux|Stable, testing, and unstable|rsync://rsync.debian.org/debian/|rsync|~2.0 TiB|large|12h|0|0|0
```

## Alert Config (`/etc/ezmirror/alert.conf`)

Permissions are set to `600` (root-readable only).

```ini
ALERT_WEBHOOK="https://discord.com/api/webhooks/..."
ALERT_EMAIL="admin@example.com"
```

## nginx Config (`/etc/nginx/sites-available/default`)

Generated by setup.py. Key structure:

```
server {
    listen 80 default_server;
    server_name mirror.example.com;

    location /debian/ {
        alias /var/www/html/debian/;
        fancyindex on;
        fancyindex_header "/.templates/header.html";
        fancyindex_footer "/.templates/footer.html";
        fancyindex_css_href "/.templates/style.css";
    }

    location /admin/ {
        proxy_pass http://127.0.0.1:8080/admin/;
    }

    location = /healthz {
        proxy_pass http://127.0.0.1:9633/healthz;
    }

    location = /status.json {
        root /var/www/html;
        add_header Cache-Control 'no-cache, no-store, must-revalidate';
    }
}
```

If Let's Encrypt certificates are found, nginx config includes TLS with HTTP to HTTPS redirect.

## Systemd Services

### ezmirord.service (Daemon)

```
[Unit]
Description=ezmirord - Mirror Sync Daemon
After=network-online.target

[Service]
ExecStart=/usr/local/sbin/ezmirord --daemon
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### ezmirror-panel.service (Admin Panel)

```
[Unit]
Description=ezmirror Admin Panel (FastAPI)
After=network-online.target nginx.service

[Service]
ExecStart=/usr/local/bin/ezmirror-panel
Restart=always
RestartSec=5
User=nobody
Group=nogroup

[Install]
WantedBy=multi-user.target
```

### ezmirror-sync.service + .timer (Periodic Sync)

```
[Unit]
Description=ezmirror Sync

[Service]
Type=oneshot
ExecStart=/usr/local/sbin/ezmirord --sync

[Timer]
OnBootSec=5min
OnUnitActiveSec=6h
Persistent=true
```

## Fancyindex Templates (`templates/`)

Three files customized during setup:

| File | Placeholders | Purpose |
|------|--------------|---------|
| `header.html` | `__LAB_NAME__`, `__DOMAIN__`, `__LOCATION__` | Top bar, breadcrumb, table header |
| `footer.html` | `__LAB_NAME__` | Sorting JS, search, icons |
| `style.css` | none | Dark/light theme, responsive layout |

Placeholders are replaced with values from `lab.conf` during setup.

## Admin Panel Environment

| Variable | Default | Description |
|----------|---------|-------------|
| `EZMIRROR_ADMIN_PORT` | `8080` | Panel HTTP port |
| `EZMIRROR_ADMIN_HOST` | `127.0.0.1` | Panel bind address |

These can be set to change the admin panel's listening address, but the nginx proxy config must match.
